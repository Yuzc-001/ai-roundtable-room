## 2. What "Real 质感" Actually Means in This Domain (Evidence-Based Synthesis)

High-end knowledge work / deliberation / decision support tools (Linear for issue tracking/roadmapping, advanced Figma/Notion for professional workspaces, Arc for productivity, strategy consulting workbenches, decision intelligence platforms) create "this is serious work" signal through deliberate design choices that reduce anxiety, signal reliability, and reward sustained attention — not through "AI magic" or trendy visuals.

**Synthesized from primary research (Linear redesign article + supporting sources)**:

- **Linear (gold standard for professional knowledge work UI, 2024+ redesign)**: Full article (https://linear.app/now/how-we-redesigned-the-linear-ui) details the process:
  - Core goals: "reducing visual noise, maintain visual alignment, and increase the hierarchy and density of navigation elements."
  - "Timeless UI" — not trendy or "AI-flavored"; cohesive visual language across product.
  - Technical: LCH color space for perceptual consistency (lightness/chroma/hue); limited "chrome" (borders, shadows, UI furniture used sparingly for depth/meaning only).
  - Process: Small dedicated team, stress tests (different environments, appearances, hierarchy levels), dogfooding, feature flags for rollout, milestone-based (not big-bang). Result: UI that feels "essential" — project cards show only critical info; navigation is dense but scannable; alignment (labels, icons, buttons) pixel-perfect.
  - Why it creates 质感: Users feel the product respects their time and the stakes of their work. No visual clutter competing with content. "This is for people doing real, ongoing work."

- **Figma (professional use) + related principles** (from Figma resources + UXPin analysis of Linear-like systems):
  - Design systems enforce consistency (components, spacing, typography scale) → credibility ("this was built with care and will not surprise me").
  - 8px/24px grids, strict alignment/proximity/contrast for hierarchy without noise. "Essential info only" in cards (Linear project cards example).
  - Micro-interactions purposeful (smooth, confident, not flashy or "AI-cute").
  - Breathing room balanced with density: white space for focus, but high information density where needed for experts.

- **Notion (advanced/pro use), Arc, decision platforms**:
  - Flexible structures with controlled hierarchy; spatial organization that lets users focus (Arc tabs/spaces philosophy).
  - Empty/loading/error states as "first-class citizens" — clear, actionable, professional tone (no "hello, try this demo").
  - Credibility via provenance, auditability, human control points (visible history, permissions, exports) made visually distinct.
  - Motion language: subtle, physics-like or spring-eased for "intelligence" perception (reduces "black box" anxiety, as our simPhase already attempts).

**Why these create substance/credibility (vs. toy/AI demos)**:
- Toy/AI demos prioritize "wow" (colorful avatars, streaming text, fun greetings, rounded everything, promotional starters) → signals "this is a prototype/experience."
- Professional instruments prioritize **reduction + precision + care signals**: alignment (trust in details), limited palette/chrome (focus on content), purposeful feedback (perception of reliability/intelligence), human control made visible (gates, provenance), timeless typography/hierarchy (seriousness), empty states that assume expert user ("what do you want to deliberate?").
- Result: User feels "I can stake real decisions on this" and "the makers respect the complexity of my work." Our project already has the hard parts (structured phases, human gate, project memory, evidence) that generic AI chat lacks — the upgrade is making the surface match.

Sources: Linear redesign article (primary); web_search results on Figma/Linear/Notion/Arc/professional SaaS (e.g., UXPin "Linear design explained", Figma design system resources, general "credibility signals in enterprise software").

## 5. Specific Design/UX Principles This Project Should Adopt

Tailored to local-first, structured, human-in-the-loop identity (protect these while escaping toy territory):

1. **Substance over demo**: Every visual and textual choice must signal "high-stakes, repeatable deliberation with audit trail." No cute greetings, role-play framing ("大臣"), or promotional starters unless explicitly educational and skippable.
2. **Ruthless reduction of visual noise + perfect alignment** (Linear core): Consistent 4/8px grid, pixel-perfect alignment of labels/icons/buttons, limited chrome (shadows/borders for meaning only). Audit every container.
3. **Amplify the human gate and provenance** (core differentiator): MemoryReviewPanel, citations, evidenceLabel, usage, project memory injection, undo/delete history — make these the most visually "weighty" and distinct elements. They are the credibility engine.
4. **Purposeful perception of intelligence**: Existing simPhase, phase stepper, generating HUD, focusMode are excellent — polish for confidence (subtle, not gimmicky). Feedback that reduces anxiety without "AI magic" theater.
5. **Timeless over trendy/AI-flavored**: Leverage existing serif for brand/headers (professional Chinese texture), Inter for UI. Limited, consistent palette (enforce LCH-like thinking if expanding). No excessive rounds, gradients, or per-persona color explosions unless functional.
6. **Empty/loading/error states as professional instruments**: Assume expert user. Clear next actions, no "hello digital cabinet." "What complex problem do you need to deliberate?"
7. **Immersion & focus as first-class**: Strengthen focusMode (already good) as the "serious work" mode. Three-column default for context; focus for deep sessions.
8. **Self-documenting care**: The project's strength (docs, tests, doctor, logging) should be echoed in UI (visible provenance, audit-friendly exports, clear memory flows).

These directly counter the diagnosed symptoms while preserving every protected strength.

- **Scope boundary pressure**: User may want faster "wow" or E-path. **Mitigation**: This strategy explicitly defers; reference strategic-decisions.md and A+E in all planning. Living docs prevent drift.
- **Visual QA on dark/light + mobile**: Existing responsive + theme support is good but will need re-audit post-polish. **Mitigation**: Include in verification.
- **High ROI / low downside overall**: The project already has the hard, differentiated parts (structured deliberation + human memory gate + local project persistence). This upgrade makes the surface worthy of the substance — exactly the "真正高质量升级重塑" asked for, without compromising what makes it special.
