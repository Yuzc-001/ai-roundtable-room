# 上线漏洞审查记录

## 第一轮

1. 旧版只有静态演示脚本，不能生成真实会议。
   - 修复：新增 Express 后端和 `/api/meetings`，由服务端调用模型 API。

2. 浏览器直连模型会暴露 API Key。
   - 修复：前端只访问同源后端，API Key 仅从服务端环境变量读取。

3. 浏览器 Babel、CDN React 和开发版本脚本不适合上线。
   - 修复：改为 Vite + React 生产构建，并删除旧 HTML/JS 原型入口。

4. AI 输出结构不稳定会导致前端崩溃。
   - 修复：使用 Zod 校验模型 JSON，并在服务端二次清洗 speaker、vote、citation。

5. 用户输入没有边界。
   - 修复：议题限制 800 字，补充背景限制 2000 字，JSON 请求体限制 64kb。

6. 公开接口可能被刷爆模型成本。
   - 修复：增加 IP 限流；支持 `APP_ACCESS_CODE` 访问码保护。

7. 模型可能生成危险 citation URL。
   - 修复：只允许 `http` 和 `https` 链接，其余丢弃。

8. 假纪要按钮会误导用户。
   - 修复：纪要按钮导出真实 Markdown 会议纪要。

## 第二轮

1. OpenAI 结构化输出不接受普通 optional 字段。
   - 修复：模型生成 schema 改为 required + nullable，并保留应用内部的简洁结构。

2. 角色编辑没有进入真实模型请求。
   - 修复：前端提交当前参会角色，后端只接受所选 preset 内角色的安全字段覆盖。

3. 生产服务只监听本机地址会导致容器部署不可访问。
   - 修复：生产默认监听 `0.0.0.0`，开发默认监听 `127.0.0.1`。

4. source map 会暴露源码。
   - 修复：关闭生产 sourcemap。

5. API 错误可能泄露内部细节。
   - 修复：客户端只返回通用错误，详细错误只进服务端日志。

## 第三轮

1. 访问码不应持久化在浏览器存储中。
   - 修复：改为仅保存在页面内存，刷新后需重新输入。

2. 旧样式字距在窄屏会增加文字溢出概率。
   - 修复：统一为 `letter-spacing: 0`。

## 当前结论

本轮本地审查未发现新的可直接修复漏洞。公开上线前仍需要在部署平台设置当前供应商的 API Key，并建议设置 `APP_ACCESS_CODE`，否则任何访问者都能消耗你的模型额度。

## 第四轮

1. 本地 `.env` 指引和实际启动行为不一致。
   - 修复：启动时自动读取项目根目录 `.env`，不再要求用户额外传 `--env-file`。

2. 后端从 `src/data` 读取共享数据，长期维护边界不清。
   - 修复：迁移到 `shared/personas.js`，前端通过 `src/data/personas.js` re-export 保持兼容。

## 第五轮

1. 只支持单一模型供应商会降低公开分发的可用性。
   - 修复：新增 OpenAI 兼容和 Claude 兼容供应商适配层。

2. 不同模型接口返回格式不同，容易把兼容逻辑散落到业务代码中。
   - 修复：供应商适配器只返回文本，会议层统一解析 JSON、校验和清洗。

## 第六轮

1. 访问码如果作为每次请求头发送，浏览器网络面板、代理日志或脚本复用都会增加泄漏面。
   - 修复：新增 `POST /api/sessions`，访问码只用于换取服务端签名的 HttpOnly Cookie，`POST /api/meetings` 只认会话。

2. 公开演示接口即使有分钟级限流，仍可能被长时间低频刷掉模型额度。
   - 修复：新增 `DAILY_MEETING_LIMIT`，对单 IP 增加每日生成预算。

3. 会话签名密钥如果隐式依赖访问码，不利于公开部署轮换。
   - 修复：新增 `SESSION_SECRET`，生产环境可独立轮换会话密钥。

## 第七轮

1. 单模型扮演多个角色时，形式上是圆桌，但判断来源仍然单一。
   - 修复：移除单/多模型两层模式，改为一层 `AI_PROVIDERS` 资源池；每个角色发言都从同一个资源池路由。

2. 用户可能配置 GLM、DeepSeek、OpenAI 兼容网关等五种以上 API。
   - 修复：支持任意 provider id，按 `${PROVIDER_ID}_API_KEY / BASE_URL / MODEL` 读取配置。

3. 不同角色可能需要固定供应商，例如风险官走 DeepSeek、指标官走 OpenAI、用户视角走 GLM。
   - 修复：保留 `ROLE_PROVIDERS`，支持 `角色id:供应商` 的显式路由。
