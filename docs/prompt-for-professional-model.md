### 当前项目实际情况（请你充分理解）：

- 这是一个前后端分离的应用（React 前端 + Node/Express 后端），支持本地运行和 Docker 部署。
- 目前定义了 6 个角色（Persona）：
  - 渡（du）：主持人·哲学引导者（Moderator）
  - 灼（zhuo）：远见·可能性主义者
  - 砺（li）：怀疑·风险拆解师
  - 衡（heng）：数据·证据驱动者
  - 澈（che）：共情·用户视角守门人
  - 漾（yang）：跨界·类比联想者
- 不同场景（预设）会使用不同数量的角色组合，例如：
  - 产品评审：5 个参与者 + 主持人
  - 头脑风暴：3 个参与者 + 主持人
  - 写作工坊、法律咨询等也有不同配置
- 我们目前的核心思考方向是：
  - 圆桌会议应该以“项目”为单位进行记忆管理，而不是做长期角色人格演化。
  - 每个具体项目可以反复开多场圆桌，系统需要记住该项目的上下文、历史结论、风险和行动项。
  - 我们不想做玩具，而是要做真正能产生高质量、结构化、可落地的审议结果的工具。
